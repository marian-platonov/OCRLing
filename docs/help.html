OCRLing Documentation

 Notes
This is a standalone OCR (Optical Character Recognition) tool in Python, leveraging Tesseract and Mistral AI to extract text from images. This tool identifies the languages in which the text is written and provides translation into multiple languages using Google Translate. It also includes a browser-based QR & Barcode Reader for decoding QR codes and barcodes from image files or a live camera feed.
For Support engineers, this tool significantly improves efficiency by automating the extraction and translation of error trace details from images submitted by customers. Instead of manually transcribing or interpreting text, engineers can quickly process images and work with the extracted data. This ensures faster troubleshooting and enhances customer support.
For customers/users, this tool helps bridge language barriers. If an error message appears in an unfamiliar language, the tool provides immediate translation, enabling a quicker understanding of technical issues and facilitating smoother communication with support teams.
By eliminating tedious manual tasks and enhancing language accessibility, this OCR tool streamlines workflows, improves accuracy, and accelerates case resolution.

This tool was built using Python 3.13.9 x64 bit using tkinter and customtkinter UI-libraries.

The logging feature was built using Loguru library.

The language detection feature was built using langdetect library.

The majority of icons were retrieved from Icons8.

You don't need to install Python or other libraries on your machine in order to run the OCRLing.exe application.

Some images in this documentation are zoomable. If you click on the image, you might be able to zoom out for a closer look.

Prerequisites for using this tool:

  • A Windows 64-bit operating system
  • Tesseract OCR installed on the system
  • A Mistral AI API Key (required only when using the Mistral OCR engine)
  • To have installed a file archiver for uncompressing the OCRLing_vMAJOR_MINOR_PATCH.zip file. We recommend 7-Zip.
  • Ensure the tool is stored in a local folder with Read, Write, and Execute permissions enabled for your user or group.
  • Resolutions below 1280 x 960 pixels are not supported. Running OCRLing on such displays causes the application view to be displayed incorrectly (some UI elements may not be visible).

  7. Selecting QR & Barcode Reader opens the browser-based reader in your default system browser, allowing you to decode QR codes and barcodes from image files or a live camera feed.

    8. Overview

    The QR & Barcode Reader is a browser-based tool that decodes QR codes and barcodes entirely on-device — no image data is ever uploaded to a server. It supports two input modes:

    • Image mode — decode a static image loaded from a file, drag-and-drop, or the clipboard.
    • Camera mode — decode codes in real time from a connected webcam or device camera.

    The tool uses the ZXing library for static image decoding and html5-qrcode for live camera scanning, giving reliable coverage of all common 1D and 2D barcode formats.

    Supported Formats

    • 2D codes: QR Code, Aztec, Data Matrix, MaxiCode, PDF-417
    • 1D barcodes: Code 39, Code 93, Code 128, EAN-8, EAN-13, UPC-A, UPC-E, Codabar, ITF, RSS-14, RSS Expanded, UPC/EAN Extension

    Step 1 — Open the Tool

    Select QR & Barcode Reader from the right-click tray menu.

    The page opens in Image mode by default. Two tab buttons — 🖼️ Image and 📷 Camera — appear at the top-right of the Input Source panel to switch between modes.

    The ℹ️ info button (top-right of the page) opens the user guide. The 🌙 theme button toggles between dark and light mode.

    If the ZXing library fails to load from CDN (e.g. no internet connection on first use), a warning banner appears at the top of the page. Reload the page once connectivity is restored.

    Step 2 — Load an Image (Image Mode)

    Four methods are available for loading an image into the tool:

    1. Drag and drop — Drag an image file from File Explorer and drop it onto the drop zone area labelled "Drop image here".
    2. File picker — Click the drop zone (or the 📁 Load New button after an image is already loaded) to open the system file picker. Accepted formats: PNG, JPG, GIF, BMP, WebP, TIFF.
    3. Paste (Ctrl+V) — Copy an image to the clipboard (e.g. from a screenshot tool or browser right-click → Copy image) and press Ctrl+V anywhere on the page while the Image tab is active.
    4. Paste button — Click 📋 Paste (visible after an image is loaded) to pull the current clipboard image via the browser Clipboard API. The browser will prompt for clipboard permission on the first use.

    Once loaded, a canvas preview of the image is displayed below the drop zone. The image enhancement controls and the 🔍 Scan Now button appear automatically.

    Step 3 — Enhance the Image (Optional)

    The Image Enhancement panel is shown on the right once an image is loaded. Adjustments are applied in real time and update the preview canvas. All sliders and toggles can be reset together by clicking ↺ Reset.

    • Brightness (10 – 300 %) — Lightens or darkens the image. Increase this if the code is printed on a very dark background.
    • Contrast (10 – 300 %) — Sharpens the difference between light and dark regions. Increasing contrast often helps with faded or low-quality prints.
    • Binarize (threshold 0 – 255) — Converts the image to pure black and white using the given threshold value. A threshold of around 128–150 works well for most printed barcodes. Leave at 0 (Off) to skip binarization.
    • Grayscale — Removes colour information before decoding. Recommended for coloured QR codes or codes printed on coloured backgrounds.
    • Invert — Inverts all pixel values, making dark areas light and vice versa. Useful for white-on-black or laser-engraved codes.
    • Sharpen — Applies a sharpening convolution kernel to enhance edges. Helps with slightly blurry or low-resolution images.

    Step 4 — Scan the Image

    Click 🔍 Scan Now to decode the currently displayed (and enhanced) image. A spinning indicator appears during the scan. The result is shown in the Decoded Result panel on the right.

    If no code is found, an error message is displayed with a suggestion to try enabling Grayscale, increasing Contrast, or using Binarize. Adjust the enhancement sliders and click Scan Now again — no need to reload the image.

    To remove the current image and start over, click 🗑️ Clear.

    Step 5 — Switch to Camera Mode

    Click the 📷 Camera tab to switch to live camera scanning.

    The camera does not start automatically — it only starts when the user explicitly clicks ▶ Start Camera.

    A camera selector dropdown lists all available cameras on the device. If multiple cameras are present (e.g. front and rear on a tablet), select the desired one before or after starting. Changing the selection while the camera is running switches to the newly selected device immediately.

    Step 6 — Start and Use the Camera

    Click ▶ Start Camera to request camera permission and begin scanning. The browser will display a permission prompt on the first use — allow camera access to continue.

    Once running:

    • The Active / Inactive badge next to the controls turns green and shows ● Active to confirm the camera is live.
    • The video feed fills the viewer area. A shaded scan region (dimmed border with a bright centre box) is overlaid on the video — position the QR code or barcode inside this bright area for fastest recognition.
    • The camera scans continuously at 15 frames per second until the user stops it. No manual trigger is required.
    • When a code is detected, a green highlight box appears around the detected code in the video frame.

    Click ■ Stop to stop the camera. The badge returns to ● Inactive.

    Step 7 — View the Camera Scan Result

    Each time a new code is detected, the result is shown in two places simultaneously:

    • Inline result card (below the video) — Shows the detected format badge, the full decoded text, and a thumbnail image of the captured video frame at the moment of detection. The same code is not reported again within 1.5 seconds to prevent duplicate entries.
    • Decoded Result panel (right side) — Shows the format name, character count, and the full decoded string. The 📋 Copy and 🌐 Open URL buttons are available here for post-scan actions.

    Click ✕ Clear in the inline result card to dismiss the camera scan result. This also resets the right-panel result and allows the next scan to be shown cleanly. The camera continues scanning after clearing — the camera is not stopped.

    Step 8 — Work with the Decoded Result

    After a successful scan (image or camera), the Decoded Result panel shows:

    • Format badge — colour-coded label identifying the code type: blue for QR Code, purple for 2D codes (Aztec, Data Matrix, PDF-417, MaxiCode), orange for 1D barcodes.
    • Format and length — e.g. Format: Code 128 · 12 characters.
    • Decoded data — the full plain-text content of the code.

    Available actions:

    • 📋 Copy — Copies the decoded text to the clipboard. A toast notification confirms the copy.
    • 🌐 Open URL — Visible only when the decoded text starts with http:// or https://. Opens the URL in a new browser tab.

    Controls Reference

    • 🖼️ Image tab — Switches the input panel to static image mode.
    • 📷 Camera tab — Switches the input panel to live camera mode.
    • Drop zone — Accepts image files by click, drag-and-drop, or Ctrl+V paste.
    • 📁 Load New — Opens the file picker to replace the current image.
    • 📋 Paste — Reads an image from the clipboard (requires browser permission).
    • 🗑️ Clear (image) — Removes the current image and resets the result panel.
    • 🔍 Scan Now — Decodes the currently displayed (enhanced) image.
    • ↺ Reset — Resets all enhancement sliders and toggles to their defaults.
    • Camera selector — Dropdown listing all detected cameras; used to choose input device.
    • ▶ Start Camera — Requests camera permission and begins continuous live scanning.
    • ■ Stop — Stops the camera stream and returns to the inactive state.
    • ● Active / ● Inactive badge — Real-time indicator of camera running state.
    • ✕ Clear (camera) — Dismisses the inline scan result; camera continues scanning.
    • 📋 Copy (result panel) — Copies the decoded text to clipboard.
    • 🌐 Open URL (result panel) — Opens a decoded HTTPS/HTTP link in a new tab.
    • ℹ️ button (top-right) — Opens this user guide overlay.
    • 🌙 theme button (top-right) — Toggles dark / light colour theme.

    Tips for Better Scanning

    • Blurry or low-resolution images — Enable Sharpen and increase Contrast to 150–200 %. If the image is very dark, raise Brightness first.
    • Coloured QR codes — Enable Grayscale to remove colour interference before scanning.
    • White-on-black or inverted codes — Enable Invert to flip the pixel values so ZXing sees a conventional dark-on-light code.
    • Faded printed barcodes — Use Binarize at threshold 100–150 to convert the image to strict black and white, eliminating background noise.
    • Camera scanning distance — Hold the code inside the bright centre of the scan box. Move closer or further until the code fills roughly half to two-thirds of the scan box.
    • Camera focus — The tool requests continuous autofocus from the camera driver automatically. On fixed-focus webcams this step is silently skipped; ensure the camera is at an appropriate distance so the code falls within the fixed depth-of-field.
    • Lighting — Avoid direct glare on glossy labels. Diffuse, even lighting gives the best results.

    Troubleshooting

    • "The ZXing scanning library failed to load"
      • The tool requires an internet connection to download the ZXing and html5-qrcode libraries from CDN on first use. Connect to the internet and reload the page.
      • If a corporate proxy or firewall blocks CDN traffic, allow access to cdn.jsdelivr.net and unpkg.com.
    • "No QR code or barcode detected in this image"
      • Try enabling Grayscale and increasing Contrast to 150–200 %.
      • If the code is on a dark background, raise Brightness first.
      • For printed codes on plain paper, try Binarize at threshold 120–140.
      • For white-on-black or laser-engraved codes, enable Invert.
      • Ensure the full code (including its quiet zone / white border) is visible in the image.
    • "Camera access denied" or camera permission prompt does not appear
      • Open the browser's site permissions for this page and re-enable camera access, then reload.
      • In Chrome: address bar lock icon → Site settings → Camera → Allow.
      • In Edge: address bar lock icon → Permissions for this site → Camera → Allow.
      • Camera access requires a secure context (HTTPS or localhost). If the page is served over plain HTTP, the browser will block all camera requests.
    • "Failed to start camera: OverconstrainedError"
      • This error can occur if the selected camera driver does not support the requested constraints. The tool automatically retries with progressively looser constraints (non-exact device ID, then environment/user facing mode). If all retries fail, select a different camera from the dropdown.
    • Camera starts but no code is ever detected
      • Position the barcode or QR code inside the bright centre box of the scan region overlay. Codes outside this area are not decoded.
      • Ensure there is sufficient lighting — avoid scanning in dim conditions.
      • On fixed-focus webcams, check that the code is within the camera's minimum focus distance (typically 30–50 cm). Move further away if the code appears blurry in the video.
      • Try a different camera from the selector if more than one is available.
    • The same code is detected repeatedly every few seconds
      • This is expected behaviour — the same code is intentionally suppressed for 1.5 seconds between reports. Move the code out of frame, then back in, to force a fresh detection immediately.
    • The camera dropdown is empty or shows "No cameras detected"
      • The dropdown is populated only after ▶ Start Camera is clicked, because camera labels require a permission grant before they are readable. Click Start Camera and grant permission — the dropdown will populate automatically.
      • If no cameras appear after permission is granted, check that the camera is physically connected and recognised by the operating system (Device Manager on Windows).
    • Paste (Ctrl+V) does not load an image
      • Ctrl+V paste is only active when the Image tab is selected.
      • Ensure the clipboard actually contains an image, not a file path. Copy the image content itself (e.g. right-click → Copy image in a browser, or use a screenshot tool).
      • Some browsers limit clipboard image access; use the 📋 Paste button as an alternative — it uses the Clipboard API with an explicit permission prompt.
    • Open URL button does not appear after scanning a QR code
      • The button is only shown when the decoded text begins with http:// or https://. If the QR code contains a URL in another format (e.g. no protocol prefix), copy the text manually and paste it into the browser address bar.

How to Retrieve a Mistral AI API Key: Step-by-Step Guide

This key will allow your application to interact with Mistral AI's services for OCR and other functionalities.
  1. Visit the Mistral AI Website – Open your preferred web browser and go to the official Mistral AI website.
  2. Log in or Sign Up – If you already have an account, log in. Otherwise, create a new account by following the registration process.
  3. Navigate to API Keys – Once logged in, go to the API Keys section or developer settings in your account dashboard.
  4. Create new key – Locate the option to create a new API key, then follow the prompts to generate one.

  6. Verify API Access – Ensure the API key functions correctly by testing it according to Mistral AI's authentication guidelines. You can also verify the same key in the OCRLing Settings page by entering the Mistral AI API Key and selecting Test API Key.

Troubleshooting

QR Code and Barcode Scanner

  • QR code / barcode is not detected
    The scanner applies multiple image preprocessing algorithms automatically, but some images may be too blurry, low-contrast, or partially obstructed for reliable detection.
    Solution:
    • Use a higher-resolution or better-lit source image.
    • Ensure the QR code or barcode is not rotated more than 45 degrees from horizontal.
    • Crop the image so the code occupies a larger portion of the frame before scanning.
    • Try loading the image from file (Select Image) instead of a screen capture for better quality.
  • Scan results window appears too small or content is cut off
    Solution: Use the vertical scrollbar on the right side of the results window to scroll through all content. You can also resize or maximize the window — all content remains accessible.

General

  • OCRLing.exe does not start or crashes silently
    Solution:
    • Check the ./logs/application.log file for error details immediately after the crash.
    • Ensure the OCRLing folder is in a location where your user account has Read, Write, and Execute permissions.
