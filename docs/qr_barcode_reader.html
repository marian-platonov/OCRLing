🔍 QR & Barcode Reader — User Guide

What is this tool?

A fully local, privacy-first QR code and barcode reader that runs entirely in your browser. No images or data are ever uploaded to any server — all processing happens on your device. It supports a wide range of code formats and includes image enhancement features to improve detection on difficult images.

Why is it helpful?

🔗 Inspect QR codes before scanning with your phone

Paste a screenshot or image to reveal the URL or data hidden in a QR code — useful for security checks before visiting unknown links.

📦 Product & inventory barcodes

Read product barcodes (EAN-13, Code 128, UPC-A) from photos or screenshots for inventory management, part identification, or data entry without a physical barcode scanner.

🤖 UiPath document automation

Extract data from barcodes present in document screenshots or captured images within UiPath automation workflows — paste the image directly to get the encoded value.

📷 Live scanning station

Use your webcam or document camera as a real-time barcode scanner. Useful for quick lookups during physical inventory counts or conference check-ins.

How to use

Option 1 — Load an image file

  • Click the drop zone or the Load Image button to open a file picker
  • Select any image (PNG, JPG, GIF, BMP, WebP, TIFF)
  • The tool scans automatically and shows the result in the right panel

Option 2 — Paste from clipboard (Ctrl+V)

  • Copy an image to your clipboard: PrtScn, Snipping Tool, right-click → Copy Image, or copy from a web page
  • Press Ctrl+V anywhere on the page — the image will be loaded and scanned instantly
  • Alternatively, click Paste from Clipboard

Option 3 — Drag & Drop

  • Drag any image file from File Explorer and drop it onto the drop zone

Option 4 — Live Camera

  • Click the Camera tab at the top of the input panel
  • Click Start Camera — the browser will ask for camera permission. Allow it
  • If you have multiple cameras (built-in webcam, USB camera, document scanner), select the desired one from the dropdown
  • Point the camera at a QR code or barcode — it decodes in real time, continuously
  • The last detected code is always shown in the result panel
  • Click Stop when done to release the camera

Image Enhancement

The enhancement panel appears automatically after loading an image. If a code is not detected, try these adjustments — the image re-scans automatically after each change:

  • Brightness — raise for dark images, lower for overexposed/washed-out images
  • Contrast — increase to make the code stand out from its background (try 150–200%)
  • Binarize (threshold) — converts the image to pure black & white at a cutoff value. This is the most effective enhancement for QR codes. Try values between 100 and 160
  • Grayscale — removes color to reduce noise — often the first thing to try
  • Invert — swaps black and white — useful for light-on-dark or negative codes
  • Sharpen — applies a sharpening convolution filter — helpful for blurry or low-resolution images

Click Scan Now to force an immediate re-scan without waiting for the debounce delay. Click Reset to undo all enhancements.

Supported code formats

2D codes:

QR Code Data Matrix PDF-417 Aztec MaxiCode

1D barcodes:

Code 128 Code 39 Code 93 EAN-13 EAN-8 UPC-A UPC-E Interleaved 2/5 (ITF) Codabar RSS-14 RSS Expanded

Troubleshooting

💡 Quick fix recipe: Enable Grayscale → increase Contrast to 180% → set Binarize to 130. This resolves the majority of difficult scans.

  • "No QR or barcode detected": Try the quick fix recipe above. Also check that the entire code is visible and not cut off
  • Blurry image: Enable Sharpen, then increase Contrast
  • Dark or underexposed image: Increase Brightness to 150–200%
  • White code on dark background: Enable Invert
  • Noisy/colorful background: Enable Grayscale, then Binarize
  • Small barcode in large image: Crop the image to just the barcode area before loading
  • Camera permission denied: Go to your browser's site settings, find this page, and allow camera access, then reload
  • Clipboard paste not working: Use Ctrl+V instead of the button. Clipboard API requires HTTPS or localhost in some browsers
  • Library not loading: The tool requires an internet connection on first use to load the ZXing decoder library from CDN

Privacy

This tool runs entirely in your browser. No images, camera frames, or decoded data are transmitted to any external server. Your data never leaves your device.
